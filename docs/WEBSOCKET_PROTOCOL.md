# WebSocket Protocol

The Chat Platform uses Django Channels for real-time WebSocket communication.
Connection URL: `ws://<domain>/ws/chat/<conversation_id>/`

## Authentication
WebSockets are authenticated using JSON Web Tokens (JWT).
- **Production**: The client must include the token in the `Authorization: Bearer <token>` header.
- **Development**: The client may append the token to the query string `?token=<token>`.

---

## Client-to-Server Events (Send)
The client sends JSON objects with a mandatory `action` key.

### 1. Sending a Message
Used to send a text message to the conversation.
```json
{
  "action": "send_message",
  "text": "Hello, world!",
  "temp_id": "uuid-or-timestamp-string"
}
```
*Note: `temp_id` is an optional unique string generated by the client to support optimistic UI updates.*

### 2. Typing Indicators
Sent when the user starts or stops typing.
```json
{
  "action": "typing_start"
}
// or
{
  "action": "typing_stop"
}
```

### 3. Message Read Receipts
Sent when the client views a specific message.
```json
{
  "action": "message_read",
  "message_id": 123
}
```

---

## Server-to-Client Events (Receive)
The server sends JSON payloads back to the client(s).

### 1. Message Acknowledgment (Optimistic UI)
Sent **only to the sender** immediately after the server saves the message to the database. The client should use `temp_id` to replace the optimistic message bubble with the official `message_id`.
```json
{
  "action": "message_ack",
  "temp_id": "uuid-or-timestamp-string",
  "message_id": 124,
  "created_at": "2026-07-09T12:00:00Z"
}
```

### 2. Receiving a Message
Broadcasted to all participants in the conversation when a message is successfully sent.
```json
{
  "action": "receive_message",
  "message_id": 124,
  "text": "Hello, world!",
  "sender_id": 5,
  "created_at": "2026-07-09T12:00:00Z"
}
```

### 3. Typing Status
Broadcasted to other participants when someone starts/stops typing.
```json
{
  "action": "typing_start", // or "typing_stop"
  "sender_id": 5
}
```

### 4. Message Read Status
Broadcasted to the conversation when someone reads a message.
```json
{
  "action": "message_read",
  "message_id": 123,
  "reader_id": 5
}
```

### 5. User Presence
Broadcasted when a user connects or disconnects from the conversation.
```json
{
  "action": "user_online", // or "user_offline"
  "user_id": 5
}
```
